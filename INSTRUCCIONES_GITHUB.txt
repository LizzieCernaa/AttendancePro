========================================
INSTRUCCIONES PARA SUBIR A GITHUB
========================================

⚠️ IMPORTANTE: CAMBIA TU CONTRASEÑA DE GITHUB INMEDIATAMENTE
https://github.com/settings/security

========================================
OPCIÓN 1: CREAR REPOSITORIO VÍA WEB
========================================

1. Ir a https://github.com/new

2. Configurar el repositorio:
   - Repository name: AttendancePro
   - Description: Android app for teacher attendance management with offline support
   - Public o Private: Selecciona según tu preferencia
   - ❌ NO marcar "Add a README file" (ya lo tenemos)
   - ❌ NO marcar "Add .gitignore" (ya lo tenemos)
   - ❌ NO marcar "Choose a license" (ya lo tenemos)

3. Click en "Create repository"

4. En la página que aparece, COPIAR el URL que dice:
   https://github.com/LizzieCernaa/AttendancePro.git

5. Volver a Git Bash y ejecutar estos comandos (uno por uno):

   cd /c/Users/cerna/AndroidStudioProjects/AsisteDocente

   git remote add origin https://github.com/LizzieCernaa/AttendancePro.git

   git branch -M main

   git push -u origin main

6. Te pedirá autenticación. Tienes 2 opciones:

   OPCIÓN A - Personal Access Token (RECOMENDADO):
   - Ve a: https://github.com/settings/tokens
   - Click en "Generate new token" → "Generate new token (classic)"
   - Dale un nombre: "AttendancePro"
   - Selecciona scopes: ✅ repo (todos los permisos de repo)
   - Click "Generate token"
   - COPIA el token (ejemplo: ghp_xxxxxxxxxxxxxxxxxxxx)
   - Cuando git push te pida password, pega el TOKEN (no tu contraseña)

   OPCIÓN B - GitHub Desktop:
   - Descargar GitHub Desktop: https://desktop.github.com/
   - Iniciar sesión
   - File → Add Local Repository
   - Seleccionar: C:\Users\cerna\AndroidStudioProjects\AsisteDocente
   - Click "Publish repository"

========================================

========================================
¿PROBLEMAS?
========================================

Error "repository not found":
→ Verifica que el repositorio existe en https://github.com/LizzieCernaa/AttendancePro

Error "authentication failed":
→ Usa un Personal Access Token en lugar de contraseña
→ O usa GitHub Desktop

Error "permission denied":
→ Verifica que estás usando el usuario correcto
→ Usa Personal Access Token con permisos de 'repo'
